# 📈 AI-Driven Economic Research Lab

Welcome to the **`data-analysis` workspace**, a specialized, modular, and AI-driven economic research and forecasting environment. This project is structured as a premium research workspace, allowing human economists and AI coding assistants to collaborate seamlessly.

This workspace supports automated data acquisition, time-series transformations (including seasonal adjustments via X-13ARIMA-SEATS), econometric forecasting, modern predictive modeling, localized visual reporting, and publication-ready Markdown generation.

## 🗺️ Workspace Infrastructure

The workspace operates on a strict **Modular Directory Standard**. Inputs, intermediates, and outputs are dynamically routed to keep assets perfectly organized:

```
data-analysis/
├── .cursorrules           # Chief Economist system mandates and conventions for AI coding assistants
├── .env.example           # Non-sensitive template for API key configs
├── README.md              # This master developer and AI onboarding manual
├── assets/                # Visual assets, logos, and custom localized typography
├── bin/                   # Compiled binaries (e.g., x13as.exe for seasonal adjustment)
│   └── check_env.ps1      # PowerShell launcher for OneDrive venv resilience
├── input/                 # Raw manual spreadsheets and user-supplied CSV files
├── output/                # Consolidated output storage
│   ├── chart/             # Standard, professional static visualizations (PNG)
│   │   └── html/          # Interactive charting dashboards (HTML)
│   ├── data/              # Workspace datasets (including raw API outputs)
│   │   ├── transformed/   # Cleaned, resampled, or seasonally adjusted series
│   │   └── forecast/      # Predicted series generated by classical/ML models
│   ├── model/             # Diagnostic outputs, text summaries, and estimation logs
│   └── report/            # Publication-grade Markdown reports and audits
├── src/                   # Python application code
│   ├── api/               # API Clients (CEIC, BOT, EIA, PortWatch, and Venv Resilience)
│   ├── data/              # Resampling, cleaning, and seasonal adjustment pipelines
│   ├── visualization/     # Styling engines, chart utilities, and font localizers
│   └── validate_env.py    # Environment diagnostic and sanity checking suite
└── temp/                  # Session-specific temp files and logs (ignored in Git)
```

## ⚙️ Environment Setup & Resilience

This workspace is designed to sync seamlessly via OneDrive across different machine architectures and usernames. If virtual environment paths break due to directory differences, run the automated resilience utility to repair it.

### Step 1: Clone and Provision Configuration
1. Duplicate `.env.example` and rename it to `.env` in the project root.
2. Fill in the required API keys for CEIC Database, Bank of Thailand (BOT), and U.S. EIA Energy indicators.

### Step 2: Run Environment Resilience Check
Run the launcher script to automatically verify and align username-specific paths in the virtual environment configuration (`pyvenv.cfg`):
```powershell
powershell -File .\bin\check_env.ps1
```

### Step 3: Run the Diagnostics Suite
Confirm that all 12 third-party analytical libraries (Pandas, Statsmodels, Matplotlib, etc.) are available, local SQLite database engines are healthy, and the seasonal adjustment binary is fully functional:
```powershell
powershell -Command "$env:PYTHONPATH='.'; .\.venv\Scripts\python.exe src/validate_env.py"
```
*A successful diagnostic run will print a clean green status audit table of all workspace modules.*
