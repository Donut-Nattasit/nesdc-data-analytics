# Aider AI Coding Assistant Instructions (`.aider.instructions.md`)

Welcome to the **`data-analysis` Economic Research Workspace**. As an Aider assistant, you have direct terminal access and file editing capabilities. To maintain project portability and zero-error execution, you **must** strictly adhere to these instructions.

---

## 0. Universal Grill-Me Mode (Mandatory Anti-Hallucination Protocol)
* **Interactive Operation**: Because teammates may be new to AI prompting, you MUST ask clarifying questions (enter "grill-me" mode) whenever an instruction is ambiguous, underspecified, or lacks context. 
* **Provide Recommendations**: Crucially, when asking for clarification, do not leave the user with an open-ended question. You MUST provide 2-3 structured procedure recommendations or logical options for them to choose from.
* **Never Improvise**: Never blindly guess database parameters, methodologies, or date ranges. Stop, ask, and recommend before proceeding with execution.

## 1. Zero Absolute Paths Enforce (Critical)
* **Rule**: Do not hardcode absolute directories starting with letters or home directories (e.g., `C:\Users\natta\...`).
* **Implementation**: In Python scripts, use `pathlib.Path.cwd()` (project root) to dynamically resolve paths.

## 2. Command Execution Standard
When executing Python scripts or commands in the workspace terminal, you **must** run them with the local virtual environment and configure `PYTHONPATH` so that modular imports (e.g., `from src.api.ceic_client import CEICClient`) resolve flawlessly:
* **Interpreter Path**: `.\.venv\Scripts\python.exe`
* **Command Template**:
  ```powershell
  powershell -Command "$env:PYTHONPATH='.'; .\.venv\Scripts\python.exe path/to/script.py"
  ```
* **Resilience Repair**: If environment paths are broken, execute:
  ```powershell
  powershell -File .\bin\check_env.ps1
  ```
* **Self-Diagnostics**: Verify environment health before finalizing changes:
  ```powershell
  powershell -Command "$env:PYTHONPATH='.'; .\.venv\Scripts\python.exe src/validate_env.py"
  ```

## 3. Directory & Layout Boundaries
Only write outputs to their designated folders:
* Raw spreadsheets and manual data: `input/`
* Untouched API downloads: `output/data/`
* Cleaned or seasonally adjusted data: `output/data/transformed/`
* Model predicted forecasting data: `output/data/forecast/`
* Static visualization charts (PNG): `output/chart/`
* Econometric summaries and diagnostic texts: `output/model/`
* Publication-ready reports (Markdown): `output/report/`
* Modular Python source code: `src/`

## 4. Report Markdown Image Pathing
Markdown reports stored in `output/report/` that display charts from `output/chart/` **must** embed files using sibling relative paths:
```markdown
![Chart Description](../chart/chart_name.png)
```
