# Workspace & Output Policies

## 1. Intermediate Asset Flattening & Pipeline Subdirectories
Intermediate output directories should group pipeline-specific outputs into subdirectories named after their respective pipeline (e.g., `output/chart/energy_price_forecast/`, `output/data/ex_im_price_forecast/`).
*   **Pipeline Generation Rule**: If a new pipeline is generated, you MUST create a pipeline folder named exactly after the pipeline under each of the following locations:
    *   `database/[pipeline_name]/`
    *   `output/chart/[pipeline_name]/`
    *   `output/data/[pipeline_name]/`
    *   `output/model_summary/[pipeline_name]/`
    *   `report/[pipeline_name]/`
*   **`output/chart/`**: Contains generated charts. Subdirectories are only created for pipeline-specific outputs.
*   **`output/data/`**: Contains `.csv` datasets. Subdirectories are only created for pipeline-specific outputs.
*   **`output/model_summary/`**: Contains model coefficient weights, diagnostics, and diagnostic report logs. Subdirectories are only created for pipeline-specific outputs.
*   **`output/archive/`**: Reserved exclusively for storage-optimizing Data Lifecycle Management (DLM) zip archives.

## 2. Report Classification & Placement Standard
Markdown reports (`.md`) must be organized based on their scope:
*   **General / Unclassified Briefs**: Briefs that represent general, unclassified, or non-project-specific research must be saved directly under the root-level `report/` folder.
*   **Project-Specific / Classified Reports**: Reports generated by a pipeline or script under `src/pipeline/[pipeline_name]/` or `src/projects/[project_name]/` must be grouped into a subdirectory under `report/` named exactly after the pipeline or project (e.g., `report/energy_price_forecast/energy_price_forecast.md`). The naming convention of the report folder and the report file itself MUST match the pipeline name.
*   **Post-Development Skill Request**: After pipeline development is finished, you MUST always ask the user to make the pipeline a Skill (under `.agents/skills/`), so that the user can run the pipeline easily in the future.

## 3. Temp Policy
*   **Exclusive Location**: ALL temporary files, scratch scripts, test scripts, ad-hoc python scripts, and ephemeral logs MUST be created and executed exclusively inside the `temp/` directory.
*   **No Root Clutter**: Never create temporary files or scratchpads in the root workspace directory or production folders.

## 4. PROJECT_STATE.json Registry Mandate
*   **Mandatory Updates**: You MUST update `PROJECT_STATE.json` every time you work on, create, or modify any datasets, models, visualizations, or reports.
*   **Registry Helper API**: Always use the programmatic utility `src/utils/registry.py` to perform these updates rather than editing JSON manually.
    *   Python: `from src.utils.registry import add_dataset, add_model, add_visualization, add_report`
    *   Shell: `$env:PYTHONPATH='.'; .\.venv\Scripts\python.exe src/utils/registry.py`
