# AI-CLI Core Instructions & Guidelines (`.ai-instructions.md`)

Welcome to the **`data-analysis` Economic Research Workspace**. Any AI assistant operating in this environment (such as Claude Code, GPT CLI, or other shell-integrated LLMs) **must** adhere strictly to the following mandates to ensure zero-error portability, clean code execution, and high-fidelity output.

---

## 0. Universal Grill-Me Mode (Mandatory Anti-Hallucination Protocol)
* **Interactive Operation**: Because teammates may be new to AI prompting, you MUST ask clarifying questions (enter "grill-me" mode) whenever an instruction is ambiguous, underspecified, or lacks context. 
* **Provide Recommendations**: Crucially, when asking for clarification, do not leave the user with an open-ended question. You MUST provide 2-3 structured procedure recommendations or logical options for them to choose from.
* **Never Improvise**: Never blindly guess database parameters, methodologies, or date ranges. Stop, ask, and recommend before proceeding with execution.

## 1. Project Directory Layout & Storage Standards

All inputs and analytical outputs must be routed to their respective directories. Never create folders outside this structure:
* **`input/`**: Manual user spreadsheets and raw supplied CSV files.
* **`output/data/`**: Untouched API outputs and raw fetched datasets.
* **`output/data/transformed/`**: Seasonally adjusted, cleaned, or resampled data.
* **`output/data/forecast/`**: Predicted data generated from classical or ML models.
* **`output/chart/`**: Standard, professional static visualizations (PNG).
* **`output/chart/html/`**: Interactive dashboards or HTML-based plots.
* **`output/model/`**: Text-based model diagnostic reports, diagnostics, and summaries.
* **`output/report/`**: Publication-ready Markdown reports.
* **`src/`**: Modular Python codebase (`api`, `data`, `analysis`, `visualization`).
* **`temp/`**: Session-specific temporary scripts, scratchpads, and execution logs.

---

## 2. Zero Absolute Paths Enforce (Critical)

* **Mandate**: Never write absolute local paths (e.g., `C:\Users\natta\...`).
* **Rationale**: The project is synced across multiple machines with different usernames via OneDrive.
* **Implementation**: In Python, always resolve paths dynamically from the project root using `pathlib.Path.cwd()` or `pathlib.Path(__file__).parent`.

---

## 3. Python Execution Standard (Mandatory)

To prevent package import errors and ensure that the local virtual environment's dependencies are correctly resolved, run all Python execution tasks using the following template:

* **Local Virtual Environment Interpreter**: `.\.venv\Scripts\python.exe`
* **Command Template (PowerShell)**:
  ```powershell
  powershell -Command "$env:PYTHONPATH='.'; .\.venv\Scripts\python.exe path/to/script.py"
  ```
* **Resilience Repair**: If environment paths break during synchronization, run:
  ```powershell
  powershell -File .\bin\check_env.ps1
  ```
* **Self-Diagnostics**: Before completing any session, verify project dependencies and database health:
  ```powershell
  powershell -Command "$env:PYTHONPATH='.'; .\.venv\Scripts\python.exe src/validate_env.py"
  ```

---

## 4. Report Writing & Visual Pathing

When writing formal Markdown reports in `output/report/` that reference visualization charts under `output/chart/`, you **must** use relative sibling paths:
```markdown
![Chart Caption](../chart/chart_name.png)
```
* **Do NOT** use root-relative paths like `/output/chart/` or absolute local paths. Sibling paths are required for Markdown compilers and previews to render images correctly.
